# Room Capture for 3DGS (AnySplat)

スマホのブラウザから**部屋を撮影**し、**AnySplat 用の高品質な画像セット(48〜60枚)を能動的に集める**ためのガイド付きキャプチャ Web アプリ。GitHub Pages（静的ホスティング）だけで動作し、**画像は端末外に送信されません**。

## コンセプト

「たくさん撮らせる」のではなく、**良い 48〜60 枚を能動的に集める**。核は次の3点:

1. **自動シャッター(視差バンド)** — 前回採用フレームからのカメラの動き量(映像＋ジャイロ)が一定量を超えたら自動で1枚採用。止まっている冗長区間は撮らず、動いた分だけ視点が離れた（＝視差の大きい）フレームを集める。
2. **カバレッジ・ドーム可視化** — 方位を16分割したリングを表示。各方向を規定枚数撮ると緑に。**部屋の全方向(壁・奥・隅)を満遍なく**埋めるよう誘導。
3. **ブレ・パン検知** — ラプラシアン分散でブレを検知して見送り。ジャイロ角速度が大きすぎると「ゆっくり」警告。その場回転（パン）ばかりのときは「歩いて視点を変えて」と促す。

さらに撮影後、**端末内のセーフティネット選別**を必ず実行:
- **ブレ除去**（シャープネス中央値の 0.5 倍未満を除外）
- **視点均等化**（累積動き量で等間隔に間引き、最大60枚に整える）

これは AnySplat 検証時にサーバー側で有効だった「B選択」を、そのままブラウザ内(オンデバイス)で再現したもの。

## 使い方（スマホ）

1. スマホのブラウザで公開URL（GitHub Pages）を開く
2. 「撮影を始める」→ カメラとモーションセンサーを許可
3. **横向き**にして、部屋を**ゆっくり歩いて回り込み**ながら撮る
4. リングの灰色の方向を埋める。48枚以上＆全方向が緑になれば十分
5. 「撮影終了」→ 端末内で選別 →「zip をダウンロード」

## AnySplat への渡し方

zip を解凍すると `frame_0001.jpg …` と `capture_meta.json`（各フレームの方位・シャープネス・時刻）が入っています。これらを AnySplat パイプラインの入力フレームとして使えます（**ブレ除去・視点均等化は端末側で済み**なので、そのまま推論に投入可能）。

## 技術メモ

- 依存ライブラリなし・ビルド不要のバニラ JS。外部CDN・外部通信なし（GitHub Pages / CSP に優しい）
- センサー: `DeviceOrientation`（方位）＋ `DeviceMotion`（加速度・角速度）。iOS は初回に権限リクエストが必要
- 動き/ブレ判定は 96×54 のグレースケール縮小画像で軽量に実施
- zip は無圧縮(store)＋CRC32 を内蔵実装（JPEG は既圧縮のため store で十分）
- **HTTPS 必須**（`getUserMedia` / DeviceMotion の要件）。GitHub Pages は HTTPS

## 調整パラメータ

`js/app.js` 冒頭の `CFG` で調整できます:

| 定数 | 意味 |
|---|---|
| `TARGET_TOTAL` / `MIN_TOTAL` | 目標／最低採用枚数 |
| `SECTORS` / `MIN_PER_SECTOR` | 方位分割数／各方位の最低枚数 |
| `MOTION_BAND` | 自動シャッターの視差しきい値（大きいほど視点を離す） |
| `BLUR_REL` | ブレ判定の相対しきい値 |
| `FAST_ROT_DPS` | 「速すぎ」警告の角速度 |

## 制限

- feed-forward 3DGS は VRAM 依存のため、AnySplat 側の入力は 40〜60 枚程度が安全（本アプリの既定はこの範囲）
- 縦持ちは AnySplat の中央クロップで視野を失うため、横持ちを強く推奨（アプリも横向きを促す）
- DeviceMotion/Orientation の挙動は端末・ブラウザ差があり、方位が取れない端末ではジャイロ積分で代替
